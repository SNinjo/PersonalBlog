Root 權限

    sudo <command>
    shell
  • su (switch user): 輸入 root 的密碼取得權限並開一個新的 shell

  • 結合兩者: 輸入自己的密碼取得權限並開一個新的 shell

chmod

change mode

  • 列出檔案與資料夾的權限 
    ls -l
    shell
    total 4
    drwxr-xr-x 2 sninjo sninjo 4096 Nov 24 06:21 dir
    -rw-r--r-- 1 sninjo sninjo    0 Nov 24 06:21 file.txt
    output
    每一行中，第一個字符代表的是這個物件的型態，「-」是檔案而「d」是資料夾。 接下來的九個字符分別代表三組的權限。
    1. 使用者權限 (前面三個字符): 檔案擁有者
    2. 群組權限 (中間三個字符): 檔案群組的成員
    3. 其他人權限 (最後三個字符): 不屬於前兩個類別的其他人
    字符代表意涵:
    1. -: 沒有權限
    2. r: 讀取權限，可以開啟並查閱檔案內容
    3. w: 寫入權限，可以編輯、修改(權限)、刪除
    4. x: 執行權限，若檔案為腳本或是程式，則可以執行它
  • 修改權限
    權限設定給誰?
    1. u: User，檔案擁有者
    2. g: Group，檔案群組的成員
    3. o: Others，不屬於前兩個類別的其他人
    4. a: All，所有以上提及的人.
    如何更動權限?
    1. -: 移除權限
    2. +: 給予權限
    3. =: 設定權限 (給予權限並移除其他未提及的)
    更動哪個權限?
    1. r: 讀取權限
    2. w: 寫入權限
    3. x: 執行權限
    範例: 
    touch new_file.txt
    ls -l
    chmod u=rwx,og=w new_file.txt
    ls -l
    shell
    ...
    -rw-r--r-- 1 sninjo sninjo    0 Nov 25 10:32 new_file.txt
    ...
    -rwx-w--w- 1 sninjo sninjo    0 Nov 25 10:32 new_file.txt
    output
    給予所有人執行權限 
    chmod a+x new_file.txt
    ls -l
    shell
    ...
    -rwx-wx-wx 1 sninjo sninjo    0 Nov 25 10:32 new_file.txt
    output
    給予所有人權限時，亦可以省略「a」 
    chmod +x new_file.txt
    shell

chown

change owner

  • 指派檔案或資料夾的擁有者 
    chown <user name>:<group name>? <file path>
    shell
  • 更改目前資料夾與更深層資料夾的擁有者 
    chown -R <user name>:<group name>? .
    shell
  • 更改除了目前的資料夾以外，更深層檔案與資料夾的擁有者 
    chown -R <user name>:<group name>? *
    shell

硬碟
  • df (disk free): 列出整個系統硬碟的使用情況, 參數「-lh」使得輸出格式更容易閱讀

    du -sh
    shell

    fdisk -l
    shell
  • 將裝置掛載至檔案系統的指定資料夾上，使檔案系統可以存取裝置上的檔案如同資料夾一樣 
    mount <device name> <directory path>

systemctl

control the systemd system and service manager

  • 列出服務

    systemctl
    shell
    • 未啟動的服務
    systemctl list-units --all --state=inactive
    shell
    • 所有服務
    systemctl list-units --all
    shell
  • 啟動服務 
    systemctl start <service>
    shell
  • 停止服務 
    systemctl stop <service>
    shell
  • 重啟服務 
    systemctl restart <service>
    shell
  • 使該服務自動啟動 
    systemctl enable <service>
    shell
  • 使該服務手動啟動 
    systemctl disable <service>
    shell

curl

client URL

在不同協定下，透過 URL 發送請求、傳輸資料的工具

curl <URL> \
    -X <method written in all caps> \
    -H "<header>" \
    -d '<data>' \
    -F "<form data>" \
    --cookie "<cookie>" \
    --user-agent "<user agent>"
shell

GET JSON 範例:

curl https://www.google.com \
    -X GET \
    -H "Content-Type: application/json" \
    -d '{"status" : false, "name": "SNinjo"}' \
    --cookie "name=SNinjo" \
    --user-agent "Mozilla/5.0 (compatible; MSIE 5.01; Windows NT 5.0)"
shell

Form Data 範例 (「@」代表檔案路徑):

curl https://www.google.com \
    -X POST \
    -H "Content-Type: multipart/form-data" \
    -F "name=SNinjo&file=@img/SNinjo.png"
shell

  • 延遲執行程式一段指定時間

  • while 迴圈

    舉例來說, 每秒 ping <IP> 且顯示時間

    while true; do date && ping 61.222.241.174 -c 1 && sleep 1 && echo; done
    shell

  • 新增自訂指令

    1. 執行以下的指令 
      echo ${PATH}
      shell
      /usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/snap/bin
      output
    2. 切換目錄至上述結果的第一個路徑 
      cd /usr/local/sbin
      shell
    3. 新增檔案並以其名稱作為命令名稱 
      sudo nano PING
      shell
    4. 貼上下方的程式至檔案中
      「${*}」代表所有的命令行參數 
      #!/bin/bash
      echo 'PONG' ${*}
      shell
    5. 給予該檔案可執行的權限 
      sudo chmod +x PING
      shell
    6. 讓我們在任一路徑搭配不同的參數試試看成果吧 
      PING
      shell
      PONG
      output
      PING -p 80
      shell
      PING -p 80
      output

  • 刪除自訂指令
    移除上述步驟中的檔案
