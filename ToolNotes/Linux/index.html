chmod

change mode

  • list file and directory permissions 
    ls -l
    shell
    total 4
    drwxr-xr-x 2 sninjo sninjo 4096 Nov 24 06:21 dir
    -rw-r--r-- 1 sninjo sninjo    0 Nov 24 06:21 file.txt
    output
    On each line, The first character identifies the type of entry that is a file (-) or a directory (d). The next nine characters represent the settings for the three sets of permissions.
    1. User permissions (first three characters): For the user who owns the file
    2. Group permissions (middle three characters): For members of the file's group
    3. Other permissions (last three characters): For anyone not in the first two categories
    The letters represent:
    1. -: Permission is not granted.
    2. r: Read permissions. The file can be opened, and its content viewed.
    3. w: Write permissions. The file can be edited, modified (permissions), and deleted.
    4. x: Execute permissions. If the file is a script or a program, it can be run (executed).
  • modifies permissions
    Who we are setting permissions for?
    1. u: User, meaning the owner of the file.
    2. g: Group, meaning members of the group the file belongs to.
    3. o: Others, meaning people not governed by the u and g permissions.
    4. a: All, meaning all of the above.
    What change are we making?
    1. -: Removes the permission.
    2. +: Grants the permission.
    3. =: Set a permission and remove others.
    Which of the permissions are we setting?
    1. r: Read permission.
    2. w: Write permission.
    3. x: Execute permission.
    For example: 
    touch new_file.txt
    ls -l
    chmod u=rwx,og=w new_file.txt
    ls -l
    shell
    ...
    -rw-r--r-- 1 sninjo sninjo    0 Nov 25 10:32 new_file.txt
    ...
    -rwx-w--w- 1 sninjo sninjo    0 Nov 25 10:32 new_file.txt
    output
    We can add the execute permission for everyone. 
    chmod a+x new_file.txt
    ls -l
    shell
    ...
    -rwx-wx-wx 1 sninjo sninjo    0 Nov 25 10:32 new_file.txt
    output
    We could have achieved the same thing without the "a" in the "a+x" statement. 
    chmod +x new_file.txt
    shell

chown

change owner

  • assign the ownership of a file or directory 
    chown <user name>:<group name>? <file path>
    shell
  • change the ownership on the current directory and all directories inside it recursively 
    chown -R <user name>:<group name>? .
    shell
  • change the ownership on all the files and directories recursively, while leaving the current directory itself alone 
    chown -R <user name>:<group name>? *
    shell

curl

client URL

A tool that allows you to request and transfer data over a URL under different protocols, such as HTTP.

curl <URL> \
    -i # include the HTTP response headers \
    -f # return exit code 22 if status isn't ok \
    -X <method written in all caps> \
    -H "<header>" \
    -d '<data>'  \
    -F "<form data>" \
    -c "<cookie>" \
    -A "<user agent>"
shell

GET JSON Example:

curl https://www.google.com \
    -X GET \
    -H "Content-Type: application/json" \
    -d '{"status" : false, "name": "SNinjo"}' \
    -c "name=SNinjo" \
    -A "Mozilla/5.0 (compatible; MSIE 5.01; Windows NT 5.0)"
shell

Form Data Example ("@" indicates the file path):

curl https://www.google.com  \
    -X POST \
    -H "Content-Type: multipart/form-data" \
    -F "name=SNinjo&file=@img/SNinjo.png"
shell

  • create custom command
    1. execute the following command 
      echo ${PATH}
      shell
      /usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/snap/bin
      output
    2. change directory to the first path in the above output 
      cd /usr/local/sbin

    3. create a file and name it custom command 
      sudo nano PING

    4. paste the following code into the file
      "${*}" stores all the command line arguments as a single parameter. 
      #!/bin/bash
      echo 'PONG' ${*}
      shell
    5. add executable permissions to the file 
      sudo chmod +x PING
      shell
    6. Let's try it with different parameters on any path. 
      PING
      shell
      PONG
      output
      PING -p 80
      shell
      PING -p 80
      output

    delete the above file 
    sudo rm PING
    shell
