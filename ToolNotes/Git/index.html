Normal
  • initialize (build .git directory) 
    git init
    shell
  • clone repository 
    git clone <repository URL>
    shell
    shallow clone with last one commit and specified branch 
    git clone --depth 1 -b <branch> <repository URL>
    shell
  • push local repository to remote repository 
    git push
    shell
  • pull remote repository to local repository 
    git pull
    shell
  • displays the state of the working directory and the staging area 
    git status
    shell
  • view the changes on this branch 
    git diff <file path>
    shell

Commit
  • add files to index
    • single file 
      git add <file path>
      shell
    • all files 
      git add .
      shell
  • commit indexed files to local repository 
    git commit -m "<message>"
    shell
    Before committing, it adds all files to index. 
    git commit -a -m "<message>"
    shell
  • show commit history 
    git log
    shell
  • show full commit history (reference logs) 
    git reflog
    shell
  • revert the commit, creating a new commit 
    git revert <commit hash>
    shell
  • modify the commit message
    previous commit -> target commit
    There are two commits, and we rename "target commit".
    1. enter terminal and open the rebase of the previous commit 
      git rebase -i <previous commit hash>
      shell
    2. rewrite "pick" in the first line to "reword" 
      reword <target commit hash> <old commit message>
      

      # Rebase ...
      shell
    3. fill in the new commit name 
      <new commit message>
      

      # Please enter the commit message for your changes. Lines starting
      # with '#' will be ignored, and an empty message aborts the commit.
      ...
      shell
      Successfully rebased and updated refs/heads/<branch name>.
      output
    4. safely overwrite remote branch with this one, and it only overwrite the changes 
      git push --force-with-lease
      shell
  • modify the commit author
    previous commit -> target commit
    There are two commits, and we edit "target commit".
    1. enter terminal and open the rebase of the previous commit 
      git rebase -i <previous commit hash>
      shell
    2. rewrite "pick" in the first line to "edit" 
      edit <target commit hash> <old commit message>
      

      # Rebase ...
      shell
      Stopped at <target commit hash>...  <target commit message>
      You can amend the commit now, with
      

          git commit --amend
      

      Once you are satisfied with your changes, run
      

          git rebase --continue
      output
    3. modify the commit author 
      git commit --amend --author="<author name> \<<email>\>"
      shell
    4. fill in the new commit name 
      <new commit message>
      

      # Please enter the commit message for your changes. Lines starting
      # with '#' will be ignored, and an empty message aborts the commit.
      ...
      shell
      [detached HEAD <new commit hash>] <new commit message>
      Author: <new author name> \<<new email>\>
      Date: <new date>
      ? file changed, ? insertions(+), ? deletion(-)
      output
    5. finish modification of this commit 
      git rebase --continue
      shell
      Successfully rebased and updated refs/heads/<branch name>.
      output
    6. if you have another commit here, continue repeating steps 4 to 5
    7. safely overwrite remote branch with this one, and it only overwrite the changes 
      git push --force-with-lease
      shell
  • merge commits into one
    first commit -> second commit -> third commit
    There are three commits, and we merge "second commit" and "third commit" into one.
    1. enter terminal and open the rebase of the first commit 
      git rebase -i <first commit hash>
      shell
    2. rewrite "pick" to "fixup" except for the first line 
      pick <second commit hash> <second commit message>
      fixup <third commit hash> <third commit message>
      

      # Rebase ...
      shell
      Successfully rebased and updated refs/heads/<branch name>.
      output
    3. safely overwrite remote branch with this one, and it only overwrite the changes 
      git push --force-with-lease
      shell
  • reset HEAD to a commit 
    git reset --hard <commit hash>
    shell
  • force overwrite this branch with the specified branch 
    git reset --hard <branch>
    shell
    For example, it overwrites "master" with "develop". 
    git checkout master
    git reset --hard develop
    shell

Branch
  • list all branches 
    git branch
    shell
    including remote branches 
    git branch -a
    shell
  • create the branch 
    git branch <branch name>
    shell
  • switch the branch 
    git checkout <branch name>
    shell
    Before switching, it creates the branch. 
    git checkout -b <branch name>
    shell
  • remove the branch 
    git branch -d <branch name>
    shell
  • remove the remote branch 
    git push <remote> --delete <branch>
    shell
  • merge specified branch into this branch 
    git merge <branch name>
    shell
    For example, it merges "develop" into "master". 
    git checkout master
    git merge develop
    shell

Stash
  • list all stashes 
    git stash list
    shell
  • save uncommitted changes 
    git stash
    shell
  • reapply previously stashed changes and delete it from the list 
    git stash pop
    shell
    specify the exact stash index 
    git stash pop stash@{n}
    shell
    keep the stash in the list after reapplying it 
    git stash apply stash@{n}
    shell
  • remove stashed changes
    • the newest stash 
      git stash drop
      shell
    • all of stashes 
      git stash clear
      shell

Setting

Show the URL of remote repository

git remote -v
shell

Show the URL of remote repository

get remote set-url origin <URL>
shell

Acting on this directory

git config <parameter>
shell

Acting on this system

git config --global <parameter>
shell
  • show git config (press "q" to quit) 
    git config --list
    shell
  • set config 
    git config <key> <value>
    shell
    For Example, set the user information 
    git config user.name "<username>"
    git config user.email "<email>"
    shell
